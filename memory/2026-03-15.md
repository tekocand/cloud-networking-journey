I learned :

- VPC Endpoints:
* To connect VPC with other endpoint service/resource, for example Amazon services that are in public, without VPC Endpoint, private subnet will need NAT Gateway to reach these Amazon services.
* With VPC Endpoint, VPC can connect to the endpoint service/resource without traffic going out of the AWS network, further enhancing security and efficiency.
* There are two types of VPC Endpoints:
  - Interface endpoints
  - Gateway endpoints
 # Interface endpoints relying to AWS PrivateLink technology to create Elastic Network Interface (ENI) with a private subnet within VPC.
  - Traffic will be directed using a Route 53 DNS entry, do AWS will resolve it to the private ENI address instead of public IP Address.
 # Interface endpoints is subscription based which cost 8.9 usd / month and additional cost for traffic $0.01/GB.
 # If there are more than 4 services needed then can evaluate the cost and consider using NAT Gateway instead at 37 usd/month.
 # To ensure HA, it is advised to create and endpoint on each AZ, and for efficiency can configure the instance to use specific endpoint in the same AZ, and use the other endpoint on other AZ as redundant.
 # Interface Endpoint security use Security Groups, while Gateway Endpoint's security utilize VPC Endpoint Policies
 # After created Endpoint, specific route is added to route tables targeting to vpce.
 # EC2 instance need Endpoint Policies to access the S3/DynamoDB, create role in IAM for privileges, use S3FullAccess, then modify IAM role in EC2 Instance and set to the role.
   - Create Bucket and upload the necessary test objects/files.
 # Gateway endpoint is free to use, exclusively available for Amazon S3 and DynamoDB.
  - When endpoint created, a prefix list containing all the public address of S3 and DynamoDB will be added to VPC's route table with target to gateway endpoint, as result the traffic directed to the gateway will remains in AWS network instead of going out directly to the public IP addresses of S3 and DynamoDB.


- Flow Logs is a feature to capture information (metadata, not the actual data within the payload itself) about IP traffic going to and from network interfaces in VPC.
- Flow logs can be created for VPC, subnets, and network interface connecting to ELB,RDS,etc.

- Network Manager provides dashboard to monitor and visualization tool.

Endpoint policy can be set Allow or Deny
