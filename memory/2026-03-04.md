## Later Session - Telegram Group Connection Issues

### Problem
After OpenClaw upgrade, lost connection to Telegram group "g-personal-space". Messages from group not being received.

### Root Cause
- `groupAllowFrom` had group ID as string `"-1003840474737"` instead of number `-1003840474737`
- `ackReactionScope` was set to `"group-mentions"` instead of `"all"`
- Config changes needed full gateway restart to take effect

### Solution Applied
1. Changed `groupPolicy` from `"allowlist"` to `"open"`
2. Changed `ackReactionScope` from `"group-mentions"` to `"all"`
3. Multiple gateway restarts to apply changes
4. Group ID format corrected in `groupAllowFrom`

### Key Learnings
- `ackReactionScope` controls emoji reactions, not text replies
- Config file edits need proper JSON formatting (no trailing commas)
- Gateway restart required after config changes
- Check logs with `tail -f /tmp/openclaw/openclaw-*.log` for real-time errors

### Status
✅ Group connection restored - can now receive and send messages in group without mentions
