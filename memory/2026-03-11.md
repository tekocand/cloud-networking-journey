## Backtest Dashboard P&L Fix - 11:55
**Issue:** Backtest P&L was calculated incorrectly using price movement % instead of R-based P&L.

**Example:** A 3.33R trade with 1% risk was showing ~1.09% P&L instead of 3.33%.

**Root cause:** `calculatePnL()` function used `(exit - entry) / entry * 100` which gives price change %, not account P&L %.

**Fix applied:**
- Modified `calculatePnL()` in `backtest-dashboard/index.html`
- Now calculates: `P&L = R-multiple × risk%`
- Uses stored `risk_amount` from trade (default $250 = 1% of $25k account)
- Example: 3.33R × 1% = 3.33% P&L (correct)

**Files changed:**
- `backtest-dashboard/index.html` - Updated `calculatePnL()` function

- **Time:** ~2 hours

**Key Learnings:**
- **Security Groups:** Stateful, instance-level, allow-only rules, evaluates all rules
- **NACLs:** Stateless, subnet-level, allow+deny rules, processes in rule number order
- **Stateful vs Stateless:** SGs remember connection context (auto-allow return), NACLs don't (need explicit return traffic rules)
- **Defense in depth:** NACLs for perimeter, SGs for granular instance control
