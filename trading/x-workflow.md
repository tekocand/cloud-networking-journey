# X / Twitter Workflow

## Purpose
Use X as the main narrative engine for the trading system.
This workflow is designed for browser-assisted use when Candra is logged in.

## Best Use
Use X for:
- dominant narratives
- sentiment shifts
- sector/theme rotation
- repeated asset mentions
- smart-money-adjacent commentary
- crowd behavior / overheating detection

## Browser-Assisted Flow
### Step 1 — Login
Candra logs into X in the browser.

### Step 2 — Yue reviews one of these:
- curated lists
- saved searches
- specific accounts
- theme keywords
- cashtag searches

### Step 3 — Extract signal, not noise
Yue should summarize:
- dominant theme
- secondary theme
- who is driving the discussion
- whether the theme is crowded
- whether views are aligned or split
- whether price is confirming the narrative

### Step 4 — Convert into trading output
Yue should convert X observations into:
- narrative summary
- watchlist candidates
- warning list
- top 1 to 3 focus ideas
- no-trade warning if hype is disconnected from price structure

## X Account Buckets
Create or monitor accounts in buckets like:
- macro
- crypto narrative
- equities / sectors
- flows / smart money
- sentiment / crowd behavior
- breaking news / catalysts

## Search Bucket Ideas
Use searches like:
- ticker / cashtag mentions
- sector names
- narrative keywords
- "rotation"
- "breakout"
- "distribution"
- "accumulation"
- "unlock"
- "ETF"
- "earnings"
- "guidance"
- "flows"

## Output Structure For X Review
1. Most discussed narrative
2. Fastest growing narrative
3. Crowded narrative
4. Ignored but interesting narrative
5. Assets repeatedly mentioned
6. Whether price confirms narrative
7. Main risk of believing the crowd
8. Trading implication

## Hard Rules
- Do not dump raw tweets unless asked.
- Summarize patterns, not noise.
- Distinguish crowd hype from credible catalyst.
- If possible, separate retail-style chatter from smarter commentary.
- If X is excited but charts are weak, say so clearly.
